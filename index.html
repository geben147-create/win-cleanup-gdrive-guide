PHASE 01
현황 파악 — 어디가 무거운지 먼저 확인 실행 전 필수
INFO

INFO
Downloads 폴더별 용량 (내림차순)
어느 폴더가 가장 무거운지 파악
▼ 
Get-ChildItem "$env:USERPROFILE\Downloads" -Directory | ForEach-Object {
  $s = (Get-ChildItem $_.FullName -Recurse -File -ErrorAction SilentlyContinue | Measure-Object -Property Length -Sum).Sum / 1GB
  [PSCustomObject]@{ 폴더 = $_.Name; "GB" = [math]::Round($s,2) }
} | Sort-Object GB -Descending | Format-Table -AutoSize
INFO
C 드라이브 전체 1 GB 이상 파일 목록
용량 먹는 주범 파악
▼ 
Get-ChildItem C:\ -Recurse -ErrorAction SilentlyContinue |
  Where-Object { $_.Length -gt 1GB } |
  Sort-Object Length -Descending |
  Select-Object FullName, @{N="GB";E={[math]::Round($_.Length/1GB,2)}} |
  Format-Table -AutoSize
시간이 걸릴 수 있음. C 드라이브 전체 스캔.
INFO
Downloads 100 MB 이상 파일 목록
큰 파일만 추출 — 중복/설치파일 확인용
▼ 
Get-ChildItem "$env:USERPROFILE\Downloads" -Recurse -File |
  Where-Object { $_.Length -gt 100MB } |
  Sort-Object Length -Descending |
  Select-Object FullName, @{N="MB";E={[math]::Round($_.Length/1MB,1)}}, LastWriteTime |
  Format-Table -AutoSize

INFO

SAFE
종료된 컨테이너 일괄 삭제
Exited 상태인 모든 컨테이너 제거
▼
# Exited 컨테이너만 선택 삭제
docker rm $(docker ps -a -q -f status=exited)
실행 중인 컨테이너는 건드리지 않음. Exited 된 것만 삭제.
